# Changelog

## 1.0.5
- Страница `/rotor/releases` показывает все релизы с пагинацией (по 10 на страницу), а не только последние 10; бейдж «последний» — только у первого релиза первой страницы
- Требует ядро с per_page=100 в `GithubService::getLatestReleases`

## 1.0.4
- Переход на datetime, минимальная версия движка 14.1.0
- Новая страница «Версионирование»: semver ядра, публичный API, семантика requires, чек-лист автора модуля

## 1.0.3
- Документация по `module:sync` (заменил `module:link`): синхронизирует симлинки, перепубликует файлы секции `publish`, сбрасывает кеш модулей
- Добавлена документация секции `publish` в module.php — публикация файлов модуля в произвольные директории движка
- Обновлены команды/деплой/обновление (`rotor-commands.md`, `rotor-deploy.md`, `rotor-upgrade.md`) под `module:sync`
- Переименована Lite-сборка релиза в Upgrade-сборку (`download_lite` → `download_upgrade`, иконка и текст в releases.blade.php)
- Иконка модуля в каталоге `/rotor/modules` заменена на `fa-cube`, карточка модуля не ломается на длинных названиях
- Исправлен перенос кнопок скачивания на мобильных в карточке релиза

## 1.0.2
- Добавлен публичный каталог модулей `/rotor/modules` — список из активных реестров (`module_registries`) с поиском и сортировкой, без состояния установки; показывает все версии модуля и ссылки на скачивание. Агрегация целиком в модуле (без правок ядра), без фильтра по версии сайта
- Страница релизов: добавлена кнопка скачивания Upgrade-сборки (без `vendor`) рядом с полной, когда ассет присутствует
- Счётчик загрузок суммируется по всем ассетам релиза
- Улучшена адаптивность страницы релизов на мобильных (кнопки скачивания в ряд, ссылка на GitHub и счётчик на всю ширину, компактнее карточки)
