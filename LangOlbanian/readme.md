# LangOlbanian — модуль-язык

Добавляет в движок полноценный новый язык через механизм `publish`, не трогая ядро. Язык — «олбанский» (`ol`), он же «йазыг падонкафф»: фонетическое письмо на основе русского (аффтар, превед, фтопку).

> Языковые модули называются по схеме `Lang<Язык>` (`LangOlbanian`, `LangGerman`, `LangEnglish`…) — так они группируются и единообразно сортируются в списке модулей.

## Что публикуется

| Источник (в модуле) | Назначение (в движке) |
|---|---|
| `stubs/lang/ol` | `resources/lang/ol` — серверные (`main.php`) и JS (`main.json`) переводы |
| `stubs/flags/ol.svg` | `public/assets/flags/ol.svg` — флаг (превед медвед) для переключателя языка |
| `stubs/forum/lang/ol` | `modules/Forum/resources/lang/ol` — перевод модуля Форума (`forum::forums.*`) |

После установки язык сам появляется в переключателе (`getAvailableLanguages()` сканирует `resources/lang/*`), а `@translation` отдаёт JS-переводы из `main.json`.

## Перевод чужих модулей

Перевод Форума подмешивается в `modules/Forum/resources/lang/ol`. Публикация в чужой модуль выполняется **только если он есть на диске**: ядро (`Module::getPublishMap()`) пропускает запись `publish`, чей путь назначения ведёт в отсутствующий `modules/<X>`.

Порядок установки не важен: установка/включение любого модуля запускает полную синхронизацию (`Module::syncAll()`), которая заново публикует все активные модули. Поэтому если Форум поставят позже модуля-языка — перевод подмешается автоматически. Те же гарантии даёт `module:sync` и обновление ядра.

> Условная публикация в чужой модуль требует поддержки в ядре (проверка в `Module::getPublishMap()`, начиная с соответствующей версии движка). На ядре без этой проверки запись скопируется безусловно и создаст пустой `modules/Forum`, если Форума нет.

## Перевод

Переведены **все** языковые файлы (на базе `resources/lang/ru`) в стиле языка падонкафф — все 22 файла, набор ключей полностью совпадает с `ru`, так что фолбэк не требуется. Если в будущем в ядре появятся новые ключи, недостающие автоматически падают в фолбэк `ru` (`config/app.php` → `fallback_locale`).

Первый ключ `main.php` — `'lang' => 'Олбанскей'` — родное название языка для переключателя.

## Жизненный цикл

- **Установка/включение** — файлы копируются (`publish`).
- **Отключение/удаление** — файлы удаляются (`unpublish`).
- **Обновление ядра** — файлы перепубликуются автоматически (`module:sync`).
